# 快速开始

本文档将帮助你快速了解 URDOC 平台的核心功能和基本操作流程。

## 平台概览

URDOC 围绕 **CSTAR** 教学框架构建，包含以下核心模块：

| 模块 | 功能 |
|------|------|
| 生成病例 | AI 自动生成符合执业医师考试标准的完整结构化病例 |
| 编写故事 | 基于病例创作医疗叙事故事，体现叙事医学理念 |
| 考核理论 | 根据病例自动生成符合考试要求的选择题 |
| 互动实践 | AI 模拟患者角色，进行逼真的临床问诊训练 |
| 评估能力 | 多维度评估临床能力，持续追踪学习进度 |

此外，平台还提供多模态工具，包括图像创作、影像渲染和音频合成，拓展医学教育的表达维度。

## 典型工作流程

1. **生成病例** — 设定科室、疾病类型和难度，AI 将生成完整的结构化病例
2. **编写故事** — 基于生成的病例，创作叙事医学故事
3. **考核理论** — 根据病例自动生成选择题，检验学习效果
4. **互动实践** — 与 AI 模拟患者进行问诊对话，锻炼临床技能

## 基本操作

### 模型选择

平台支持多种 AI 模型。在页面右侧的面板中可以选择适合当前任务的模型：

- 普通对话任务可选择标准模型
- 复杂推理任务建议启用支持 reasoning 能力的模型

### 生成内容

1. 在各模块页面配置参数（如疾病类型、难度等）
2. 点击生成按钮
3. 等待 AI 处理并生成结果
4. 查看生成的内容，可重新生成或编辑

### 内容管理

生成的内容会自动保存到本地存储中，方便后续查看和管理。
