# CSTAR 教学框架

CSTAR 是 URDOC 平台的核心教学框架，代表五个关键教学实践环节，形成从理论到实践的完整闭环。

## 框架结构

```
C — Case Generation（生成病例）
S — Story Writing（编写故事）
T — Test Design（考核理论）
A — Active Practice（互动实践）
R — Rate Assessment（评估能力）
```

## 各环节说明

### 生成病例（C）

AI 根据临床医学知识库，自动生成符合执业医师考试标准的标准化病例。支持自定义科室、疾病类型、并发症等因素，生成真实感强的临床场景。

### 编写故事（S）

叙事医学是现代医学教育的重要理念。此环节基于已生成的病例，AI 自动创作以患者为主线的医疗叙事故事，帮助学习者理解疾病背后的人文关怀。

### 考核理论（T）

根据病例内容和相关知识要点，自动生成多种题型的选择题，支持自定义难度和考核重点，高效检验学习效果。

### 互动实践（A）

AI 扮演标准化的模拟患者，学习者可通过自然语言与其进行临床问诊对话。支持多种临床场景，培养病史采集和医患沟通能力。

### 评估能力（R）

多维度评估学习者的临床能力，涵盖诊断准确性、治疗方案合理性、沟通能力等，支持持续追踪学习进度与成长曲线。

## 教学闭环

CSTAR 框架设计为可循环使用的教学闭环：

1. 从病例出发（C）→ 融入叙事（S）→ 检验知识（T）→ 实践应用（A）→ 反思改进（R）
2. 评估结果（R）指导下一轮病例生成（C）的难度和重点调整

这种螺旋式上升的设计遵循医学教育从理论到实践、从认知到反思的认知规律。
